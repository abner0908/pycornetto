  56      The Cornet class exposes the Cornetto xml database 
  57       
  58      Most public methods require input in the form of a shorthand for 
  59      specifying lexical units and relations, as described below. 
  60       
  61       
  62      B{Lexical units specifications} 
  63       
  64      A specification of lexical units consists of three parts, separated by a 
  65      single colon (':') character: 
  66       
  67          1. Spelling form (i.e. a word) 
  68           
  69             This can be any string without white space 
  70              
  71          2. Syntactic category (optional) 
  72           
  73             This can be any of 'noun', 'verb' or 'adj'. 
  74              
  75          3. A sense (optional) 
  76           
  77             This is number which distinguishes the particular word sense 
  78          
  79      Examples of valid lexical unit specifications are: 
  80       
  81         - slang:noun:1 
  82         - slang:noun 
  83         - slang::1 
  84         - slang 
  85       
  86       
  87      B{Relation specifications} 
  88       
  89      A specification of a relation consists of two parts: 
  90       
  91          1. Relation name (optional) 
  92           
  93             The name of a Wordnet relation between two synsets. See the 
  94             Cornetto documentation for the available relations. If not given, 
  95             all relations are tried. The special relation "SYNONYM" holds 
  96             between all members of the same synset. The relation name is not 
  97             case-sensitive; you can use lower case. 
  98           
  99          2. Depth (optional) 
 100           
 101             A digit ('0' to '9') or the plus sign ('+'). This represents the 
 102             depth of the relations that are considered during search. In other 
 103             words, the maximal number of links allowed. If not given a default 
 104             value of 1 is used. The plus represents the system maximum 
 105             (currently 9). 
 106              
 107      A relation specification must have a name, a depth or both. Valid 
 108      relation specification include: 
 109           
 110          - HAS_HYPERONYM 
 111          - HAS_HYPERONYM1 
 112          - HAS_HYPERONYM+ 
 113          - 3 
 114          - +
